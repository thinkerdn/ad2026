# May Mobility Fleet API 導入・連携 ヒアリングシート
## 交通事業者向け 必要情報収集フォーム

---

## はじめに

本ヒアリングシートは、**May Mobility** の自動運転サービスおよびFleet APIを活用するにあたり、
交通事業者様から必要な情報を収集するためのものです。

### May Mobility サービスの特徴

May Mobilityでは、**Field Autonomy Engineers (FAEs)** がルート・停留所の設定を行うため、
Web.Auto FMSのように事業者様が直接マスタを投入する形式ではありません。
本シートは以下の2つの目的で情報を収集します。

1. **サービス設計情報** — May MobilityのFAEがルート・停留所・走行環境を設定するために必要な情報
2. **Fleet API連携情報** — 事業者様のシステムとMay Mobility Fleet APIを統合するために必要な情報

---

## マスタ階層と情報構造

```
May Mobility サービス構成
 ├─ サービスエリア（Service Zone）
 │    ├─ ジオフェンス境界（地理的サービス範囲）
 │    ├─ 停留所（Preset Stops / 乗降地点）
 │    └─ 運行ルート（FAEが設定・管理）
 ├─ 車両フリート（Fleet）
 │    ├─ 車両名（Vehicle Name / 例: "marinara", "mayble"）
 │    ├─ 車両タイプ（Toyota Sienna Autono-MaaS / WAV対応）
 │    └─ ドメイン（Domain / 車両グループ）
 └─ Fleet API連携
      ├─ 認証情報（AWS Cognito / client_id, client_secret）
      ├─ リアルタイムAPI（WebSocket / トピック購読）
      └─ バッチAPI（REST / 過去データ取得）
```

## 参考：May Mobility Fleet API 主要制約事項

| 項目 | 制約 |
|------|------|
| リアルタイムAPI接続方式 | WebSocket（購読型） |
| バッチAPI接続方式 | REST（HTTP） |
| 認証方式 | AWS Cognito クライアント認証フロー |
| 1リクエストあたりの最大トピック数 | 20トピック |
| 映像（VIDEO）の同時接続最大車両数 | 4車両 |
| バッチAPIで取得可能な過去データの上限 | 5日前まで |
| バッチAPIダウンロードリンクの有効期限 | 10分 |
| Last Activeの情報保持期間 | 7日間 |
| データ形式（映像） | Protocol Buffers（Protobuf）バイナリ |
| データ形式（テレメトリ） | JSON |

## 参考：May Mobility ロールと責任分担

| ロール | 所属 | 主な責任 |
|--------|------|----------|
| Field Autonomy Engineers (FAEs) | May Mobility | ルート設計・停留所設定・走行環境マッピング |
| Fleet Command Center (FCC) | May Mobility | リモート運行監視・緊急対応 |
| Autonomous Vehicle Operators (AVOs) | 車内オペレーター | 安全監視（初期フェーズ） |
| 交通事業者担当者 | 事業者 | サービス要件定義・地域調整・乗客対応 |
| API連携担当者 | 事業者/SIer | Fleet API・Ride-Hail API統合開発 |
