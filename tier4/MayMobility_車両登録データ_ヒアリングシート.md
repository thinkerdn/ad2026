# May Mobility Fleet API 接続設定データ ヒアリングシート
## 交通事業者向け（一問一答形式）

> **目的:** May Mobility Fleet API に接続し、車両データを取得・監視するために必要な情報を事前に整理するためのヒアリングシートです。  
> **参考:** https://docs.maymobility.com/docs/intro  
> **回答方法:** 各質問に対し、空欄または表に回答を記入してください。API認証情報はMay Mobilityの担当者（FAE）から取得してください。

---

## Fleet API 接続セットアップの全体像

May Mobility Fleet API への接続は以下の順序で行います。

```
Step 1  API認証情報の設定
         └─ Client ID / Secret → アクセストークン取得（AWS Cognito）

Step 2  車両・ドメイン情報の確認
         └─ Vehicle Name・Domain URL（May Mobilityより提供）

Step 3  リアルタイムAPI（WebSocket）のトピック設定
         └─ 購読するトピック選択（最大20件/リクエスト）

Step 4  バッチAPIの設定
         └─ 過去ログ取得・ダウンロードURL発行の設定

Step 5  データパイプライン・ストレージの設定
         └─ 受信データの保存先・フォーマット・処理パイプライン定義

Step 6  サービス設計情報の提供（FAE向け）
         └─ 運行エリア・停留所・運行時間帯の情報を May Mobility FAE に提供
```
