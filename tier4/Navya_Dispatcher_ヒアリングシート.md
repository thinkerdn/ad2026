# Navya ARMA × BOLDLY Dispatcher 導入 ヒアリングシート
## 交通事業者向け 必要情報収集フォーム

---

## はじめに

本ヒアリングシートは、フランスNavya社製自動運転バス **「NAVYA ARMA（ナビヤ アルマ）」** および
BOLDLYが提供する自動運転車両運行管理プラットフォーム **「Dispatcher（ディスパッチャー）」** を
活用した自動運転バスサービスの導入にあたり、交通事業者様・自治体様から必要な情報を収集するためのものです。

国内では **株式会社マクニカ** がNAVYA ARMAの輸入・販売・メンテナンスを担い、
**BOLDLY株式会社** がDispatcherによる運行管理・設定・遠隔監視体制の構築を支援します。

### 役割分担の概要

| 項目 | 担当 |
|------|------|
| NAVYA ARMA 輸入・販売・メンテナンス | 株式会社マクニカ |
| Dispatcher（運行管理プラットフォーム）提供 | BOLDLY株式会社 |
| 3Dマップ作成・車両セットアップ | BOLDLY株式会社 |
| 遠隔監視者認定トレーニング | BOLDLY株式会社 |
| 運転手（オペレーター）提供・トレーニング | BOLDLY株式会社 |
| 日常運行管理・遠隔監視 | 交通事業者または委託先 |

---

## マスタ階層と情報構造

```
Navya ARMA × Dispatcher 構成
 ├─ 事業者情報（Dispatcher登録）
 │    ├─ 事業者ID・名称・連絡先
 │    ├─ 従業員・オペレーター（遠隔監視者）
 │    └─ 各種許可証・資格管理
 ├─ 走行環境・車両設定
 │    ├─ 3Dマップ（LiDAR + GNSS/RTKで作成）
 │    ├─ 走行ルート（経路）
 │    ├─ プライオリティゾーン（交差点・一時停止区間）
 │    └─ 車両ID・車両設定パラメーター
 ├─ 系統・停留所・ダイヤ（GTFS-JP形式またはDispatcher直接入力）
 │    ├─ 停留所（名称・標柱ID・緯度経度）
 │    ├─ 経路（系統名・起終点・往復設定）
 │    ├─ ダイヤ（時刻表・便・運行カレンダー）
 │    └─ 運賃情報（任意）
 └─ 連携・外部システム
      ├─ GTFS-RT（リアルタイム位置情報）
      ├─ Dispatcherコネクト（API連携）
      ├─ 配車予約システム（オンデマンド対応）
      └─ 交通インフラ連携（信号協調など）
```
